# 建築作品集 - 開發者與網站維護手冊 (Site Maintenance Guide)

這份手冊專為無程式碼背景的設計師與藝術創作者撰寫。您的網站目前是一套具備最高級 **「客製化」** 與 **「圖文內容分離」** 的框架。

未來所有的內容與排版編輯，都能在這個專案資料夾 (`portfolio/`) 內輕鬆完成。您甚至可以把這份檔案當成您的工具書留著！

---

## 一、所有資料的控制中樞：`data.js`
您**絕對不需要修改 `index.html`** 就可以新增或刪減專案！所有的內容都儲存在 `data.js` 中。

- **基本資料**：在 `site`, `hero`, `about` 這些大括號中，可以直接修改文字內容字串（引號內的內容），例如更換 Email 或 IG 網址。
- **編輯 建築專案 (`works` 資料夾)** 與 **攝影集 (`images` 資料夾)**：
  這裡有極為重要的屬性對應：
  - `title`：專案大標題
  - `category`：副標題或年份
  - `image`：在首頁網格看到的「第一張封面圖」
  - `imageHover`：游標移上去時會替換的預覽圖（沒有這項就不會亮）
  - `info`: 這是一個動態列表。您可以隨便創造欄位，例如 `"Site": "Taipei"`, `"Award": "1st Prize"`。

## 二、圖文的自由排版邏輯 (The Content Array)
當您點進某個 Architecture 作品時，大畫面的排版是由 `content: []` 裡面的積木堆疊而成的。
每一次堆疊，您都可以針對這個「區塊 (Block)」設定五個變數：

```javascript
{ 
    type: "image", // 若是文字段落，請填 "text"
    src: "https://...", // 若是文字段落，請改為 text: "你想說的故事"
    width: "80%", // [重要可調] 區塊寬幅。用 100% 呈現滂薄感，用 40% 呈現留白細節。
    align: "left", // [重要可調] 靠邊方式。填寫 left (靠左), center (置中), right (靠右)。
    marginTop: "120px", // [重要可調] 與上方物件的垂直距離！數字越大，垂直留白越多。
    caption: "圖名" // [自由選填] 這樣在幻燈片全螢幕時才會顯示文字。
}
```

**排版進階技巧：疊圖與負空間**  
預設的方塊是由上往下純粹排列。但如果您想讓「文字」飄上一張「大圖」的邊角，您可以把下方文字的 `marginTop` 設為負數（例如 `marginTop: "-150px"`, `align: "right"`），這樣就能創造前衛交錯的版面。

## 三、字體排版全解析教戰 (Typography in `style.css`)
如果您想控制整個網站的一致性（例如：想把所有的小字弄得更小、把標題弄得更粗），請開啟 **`style.css`**。

CSS 控制字體的幾個核心密碼如下，您可以尋找裡面的對應區塊（如 `.hero-text-container h1` 代表首頁大標），並用滑鼠點擊調整數值：
```css
.想修改的區塊 {
    /* 1. 字體大小 (Size) */
    /* rem 是相對單位（1rem大約16px），使用 3rem 會變得非常巨大，0.8rem 是微小備註字 */
    font-size: 1.5rem;    

    /* 2. 字體粗細 (Weight) */
    /* Inter 字體支援：300(極細) -> 400(標準) -> 500(中等) -> 700(粗體) -> 900(極黑厚實) */
    font-weight: 500;     

    /* 3. 字體顏色 (Color) */
    /* #000000 為全黑，#ffffff 為純白。也可使用 #666666(深灰)、#cccccc(淺灰) */
    color: #171717;       

    /* 4. 字距 (Tracking/Letter Spacing) */
    /* 文字之間的呼吸空間。越大的留白與字距能帶出強烈的當代極簡氛圍。*/
    letter-spacing: 2px;  

    /* 5. 行距 (Line Height) */
    /* 段落中上下兩行字之間的距離，預設為 1.5 倍行高。長篇幅文章可提高到 1.8 減輕視覺壓力。*/
    line-height: 1.8;     
}
```

## 四、如何建立圖片的網址 (Hosting Images)
因為您即將開發的是一個對外開放的網頁，您不能直接告訴網站去抓您電腦的 `C:\Desktop\my-photo.jpg`（訪客的電腦裡沒有這張圖）。

**兩種最標準的做法：**
1. **使用本地相對路徑 (Relative Paths) - 最推薦！**：
   在您的 `portfolio` 資料夾裡，直接按右鍵「新增資料夾」，命名為 `assets`。把您的照片丟進去。
   接著在 `data.js` 中把原本很長的 unsplash 網址改寫為相對路徑：
   `image: "./assets/pavilion-01.jpg"`
   未來當您使用 Vercel 或 GitHub Pages 將整個資料夾一鍵上傳部署時，所有圖片就會完美跟著上雲端，超級保險！

2. **使用免費圖庫外部連結 (Img Direct Links)**：
   您可以將圖片上傳至 Imgur, Flickr 或是您的個人圖床。對著網頁上的圖點選「右鍵 -> 複製圖片位址」。**請注意結尾必須要是 `.jpg` 或 `.png` 等圖片格式而非網頁網址。**

## 五、新手常見除錯技巧 (Debugging)
- **改了資料，但網頁整個壞掉或出不來（全白）？**
  ➡️ 這 100% 代表您的 `data.js` 語法發生了小錯誤。最常見的原因是新增項目時**忘記打逗號** `,`，或者引號 `""` 沒有一左一右成對。隨時在 VS Code 裡看看 `data.js` 哪一行出現了「紅色的波浪底線」，順手補上逗號就好了。
- **如何修改亂碼特效的跳動時間？**
  ➡️ 到 `app.js` 尋找 `const totalFrames = 66;` 來修改。這代表著畫格，您修改成 `100` 或 `150` 就會跳比較久。
- **如何變更滑鼠移過去的螢光色彩池？**
  ➡️ 到 `app.js` 尋找 `const neonColors = [...]`。除了這五個顏色，您可以自由增減色票，甚至全部改成 `#ffffff` 讓它變成純白色小點。
